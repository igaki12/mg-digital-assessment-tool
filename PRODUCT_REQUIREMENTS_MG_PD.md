# MG/PD Digital Assessment Tool 要件定義書

## 1. 文書の位置づけ

本書は、現行の MG 向けブラウザアプリを基礎として、重症筋無力症（MG）とパーキンソン病 / パーキンソン症候群（PD）を対象に段階的に拡張するための現実的な要件定義書である。

- 対象: 患者本人、研究者、臨床側の評価補助
- 目的: 同一患者の経時変化を、再現性を重視して記録・比較する
- 非目的: 診断の自動化、単回測定のみでの重症度断定、臨床医評価の代替

現状コードベースは MG 中心のプロトタイプであり、本書では「現行で成立している機能」と「PD を含む次期拡張」を分けて扱う。

## 2. プロダクト方針

### 2.1 基本コンセプト

- スマートフォン / PC のカメラ・マイクを用いた非ウェアラブル計測
- 外来だけでなく自宅でも実施できる短時間・低負荷の測定
- 患者間比較よりも、同一患者内の変化検知を優先
- 数値データに加え、必要に応じて元の画像・動画・音声も保存して振り返れる設計

### 2.2 対象疾患

- 主対象: MG、PD
- 将来拡張: パーキンソン症候群、その他の神経筋疾患

### 2.3 利用環境

- 初期ターゲット: モダンブラウザ上の React SPA
- 計測端末: 患者自身のスマートフォン、タブレット、PC
- 配備形態:
  - ローカル完結モード
  - 認証付きクラウド連携モード（次期開発）

## 3. 機能要件

### 3.1 現行実装で成立している機能

以下は既存コードと整合する機能である。

| 領域 | 機能 | 現状 |
| --- | --- | --- |
| MG | 眼瞼下垂評価 | Face Landmarker による EAR ベース計測 |
| MG | 上肢挙上テスト | Pose Landmarker による腕角度の時系列取得 |
| MG / PD 共通候補 | 歩行計測 | Pose Landmarker による膝角度と近似歩行指標 |
| MG | ePRO | MG-ADL / MG-QOL15r 相当質問票 |
| 共通 | ローカル保存 | IndexedDB / LocalStorage |
| 共通 | 記録閲覧 | セッション一覧、時系列確認、動画再生 |
| 共通 | 共有導線の試作 | ダミーの医師共有 UI |

### 3.2 MG 向け必須機能

- 眼瞼下垂評価
  - 上方視タスク中の EAR 変化を記録
  - 左右差と経時変化を確認できる
- 上肢挙上評価
  - 左右腕の角度推移を記録
  - 保持時間と角度低下傾向を確認できる
- 問診票
  - MG-ADL / MG-QOL15r 相当入力
  - 過去記録との比較表示
- 症状メモ
  - 測定条件、服薬状況、自由記述を紐付けられる設計

### 3.3 PD 向け追加要件

PD 対応は、初期段階では「姿勢・歩行・音声の研究用途向け補助指標」に限定する。

- 姿勢評価
  - 体幹前屈角
  - 首下がり角
  - 側方偏位
  - 正面 / 側面のガイド付き撮影
- 歩行評価
  - 歩行時の膝角度
  - 体幹傾斜
  - 相対的な移動量・歩行リズム
- 音声評価
  - 数唱
  - 定型文音読
  - 持続母音発声
  - 音声ファイル保存と特徴量抽出

### 3.4 音声機能の要件

- 収録タスク
  - 1 から 50 までの数唱
  - 定型文音読
  - 持続母音「あー」
- 保存対象
  - 音声ファイル本体
  - タスク種別
  - 収録時刻、端末情報、ノイズ警告
- 解析対象候補
  - F0
  - intensity
  - jitter
  - shimmer
  - HNR

ただし、これらの特徴量を「すべての端末でリアルタイムかつ臨床精度で安定算出する」とは要件化しない。初期段階では、録音品質の確保と後処理可能なデータ保存を優先し、特徴量算出は段階的導入とする。

## 4. 計測フロー要件

### 4.1 基本フロー

1. 疾患プロファイルを選択する
2. 対応する測定メニューを表示する
3. カメラ / マイク位置合わせを行う
4. ガイドに従って短時間の測定を行う
5. 結果確認、保存、共有準備を行う

### 4.2 UI / UX 要件

- 高齢者でも操作しやすい大きな文字と大きな操作ボタン
- 正面 / 側面の立ち位置を示すガイドオーバーレイ
- 認識良好 / 不良を色で即時表示
- 視線を画面から外しても実施できる音声ガイド
- 実行負荷が高い自動機能より、まずは迷わない手動導線を優先

### 4.3 測定タイミング

- 単回測定
- 朝 / 夕の定時測定
- 症状増悪時のイベント記録

ただし、通知・リマインド・スケジューリング機能は現状未実装であり、次期機能として扱う。

## 5. データ要件

### 5.1 保存対象

- セッションメタデータ
- 時系列数値データ
- 動画 / 音声などの元データ
- 疾患種別
- タスク種別
- 任意メモ

### 5.2 保存方針

- 初期段階はブラウザローカル保存を基本とする
- 容量負荷の大きい元動画 / 元音声は保持件数を制限する
- 数値データは長期保持し、元メディアは必要に応じて削除可能にする

## 6. システム要件

### 6.1 フロントエンド

- React + Vite の SPA
- MediaPipe Tasks によるブラウザ内推論
- IndexedDB / LocalStorage によるローカル保存

### 6.2 バックエンド

クラウド連携を実現する場合のみ必要とする。

- 認証
- 同意記録
- メディアアップロード
- アクセス制御
- 監査ログ

### 6.3 セキュリティ

- 通信の暗号化
- 認可された利用者のみアクセス可能
- 個人情報と解析データの分離方針を明示
- 研究用途では同意取得フローを必須化

## 7. 実現性を踏まえた修正事項

以下は旧文書から修正した要点である。

### 7.1 単眼カメラによる歩行速度

自己申告の身長と 1 台のカメラだけで、臨床的に信頼できる絶対歩行速度を常に算出するのは難しい。したがって初期要件では、

- 主指標: 相対移動量、関節角度、同一条件下での経時比較
- 補助指標: 近似歩行速度

とする。

### 7.2 常時監視型の自動録画

部屋の隅に固定して自動検知・自動録画する監視形式は、プライバシー面と誤検知面の負担が大きい。初期段階では、

- 手動開始の短時間計測
- 位置合わせ完了時の自動 Ready 表示

を優先し、完全自動録画は将来検証項目とする。

### 7.3 GitHub Pages と認証付きクラウド連携の両立

静的ホスティングのみでは、医療データを扱う本格的な認証・同意・アップロード基盤は完結しない。したがって、

- ローカル完結版
- 認証付きクラウド版

を別フェーズで管理する。

### 7.4 臨床妥当性

MG-ADL、QMG、MG-QOL、MDS-UPDRS などとの相関確認は、機能要件ではなく研究・検証計画である。プロダクト要件としては、

- 臨床評価と後から照合しやすいデータ構造にする
- 計測条件を記録できる

までを必須とする。

## 8. 開発フェーズ

### Phase 1: 現行 MG プロトタイプ整理

- MG 前提の文言とデータ構造を整理
- 既存測定機能の安定化
- ローカル保存と記録閲覧の改善

### Phase 2: MG/PD 共通基盤

- 疾患プロファイル切替
- タスク定義の共通化
- データスキーマの疾患非依存化

### Phase 3: PD 計測追加

- 姿勢評価
- 音声収録
- PD 向け問診 / スコア対応

### Phase 4: 研究運用機能

- 認証と同意
- セキュアアップロード
- 医療機関向け閲覧画面
